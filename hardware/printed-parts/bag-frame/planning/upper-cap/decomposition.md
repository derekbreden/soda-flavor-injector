## Paradigm Analysis

The upper cap is entirely 2.5D prismatic. Every feature is generated by:

1. Defining a 2D profile in a workplane
2. Extruding it a fixed distance along the workplane normal

The flat body is an extruded rectangle. The ribs are rectangles extruded upward from the body top face. The snap arms are rectangular cantilevers extruded outward from the long-edge faces. The hook profiles are small triangular or stepped extrusions at the arm tips. The chamfer is a standard chamfer operation on a prismatic edge.

No feature requires a different paradigm:
- No feature is rotationally symmetric (no revolve needed)
- No feature sweeps along a curved path
- No feature requires a loft between two profiles
- No feature is a free-form surface
- The frangible bridge at each hook undercut is a designed 0.2mm interface void — it is a layer of absent material, not a separate geometric paradigm

All of this maps directly to CadQuery's workplane-sketch-extrude idiom. The step-generation standard (6-step-generation.md) defines 2.5D as "declare plane → sketch 2D → extrude." Every upper cap feature fits this definition.

## Why the Snap Arms Do Not Force Decomposition

The snap arms are the features most likely to raise a decomposition question. They could, in principle, be argued as "separate flexing elements" that warrant isolation. They do not.

The argument for decomposing them would rest on: (1) different mechanical role (flexing vs. rigid body), or (2) different geometric paradigm. Neither applies here.

**Mechanical role:** requirements.md requires that snap-fit flex direction be parallel to the build plate. With the cap printed face-down, the arms extend horizontally in the X/Y plane and flex in the X direction — parallel to the build plate. This constraint is satisfied by print orientation, not by part decomposition. There is no functional reason to separate the arms.

**Geometric paradigm:** The arms are rectangular extrusions with a hook profile at the tip. The hook profile is a small stepped or triangular extrusion — also prismatic. The root fillet is a fillet operation on a prismatic edge. The frangible bridge is a designed void (0.2mm gap between hook underside and arm body). Every element of the arm is 2.5D prismatic. There is no paradigm boundary between the arm and the body.

**Assembly consequence:** If the arms were a separate part, they would require a mechanical join to the body. The concept defines the cap-to-cradle interface as a single part pressing down. Splitting the arms into separate pieces would introduce a sub-assembly interface inside the cap that would need to be designed, toleranced, and assembled — all cost with no benefit. The vision explicitly warns against over-decomposition.

## Interface Context: Cradle Lip Rebate

From cradle-platform/parts.md, the mating feature on the cradle is a 1.2mm × 1.2mm rebate running along the lip outer face at a height of 8mm above the cradle floor. The snap arm hook (1.2mm height, 90° retention face, 30° lead-in) engages this rebate when the cap is pressed down.

This interface is described in the concept (Section 2, "Upper Caps to Cradle") and confirmed in cradle-platform/parts.md (Section 3, "Relationship to the upper cap"). The rebate is a feature on the cradle side lip outer face — a horizontal rectangular groove cut into the lip. It is a single prismatic cut in the cradle. The upper cap's hook geometry is its complementary prismatic feature.

No geometry on either side of this interface requires a paradigm beyond prismatic extrusion-and-cut.

## Composition Statement

The upper cap is a single printed part. It is modeled as one CadQuery solid:

1. Base rectangular body (extrude rectangle to 1.5mm)
2. Three longitudinal ribs added to top face (extrude rectangles upward, 5mm tall, 1.2mm wide)
3. Two transverse ribs added to top face (extrude rectangles upward, 5mm tall, 1.2mm wide)
4. Four snap arms added at long edges (extrude rectangular cantilevers 20mm outward, 2mm thick, 6mm wide)
5. Hook profiles added at each arm tip (extrude stepped/triangular hook, 1.2mm height)
6. Root fillets applied at each arm-body junction (1.0mm radius)
7. Frangible bridge voids at each hook undercut (0.2mm interface gap on hook underside)
8. Perimeter chamfer on outer face top edge (1.5mm chamfer)

All operations are boolean additions or subtractions on a single body. The part is one solid. It is printed in one instance per bag position (two prints total, same file).
